SessionControl

public protocol SessionControl
  • play(streamId:bufferDuration:maxDelay:)

    Start playing the stream with the provided id. If any previous stream was playing, that stream will be stopped

    Available means, that it is in the internal list, it could be just been removed as the streamer decided to go offline, this is also propagated throug the delegate. Only after starting a stream, advertisement plays are received via the delegate

    Remark

    After the connection to the stream was established audio data is send via the MycrocastDelegate and an optional ad play can happen if the last ad was some time ago

    Declaration

    Swift

    func play(streamId: Int, bufferDuration: Int?, maxDelay: Int?) -> Bool

    Parameters

    streamId

    The id of the LiveStream to play
    bufferDuration

    The duration in ms that audio package will be buffered before playing (default 1000ms)
    maxDelay

    The duration in ms that audio packages are collected for the maximum delay (how far in the past you can get) - defaults to 1000 * 60 * 5 (5 min)

    Return Value

    True if the stream was still available false otherwise
